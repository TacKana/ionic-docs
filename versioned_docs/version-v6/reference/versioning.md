# 版本控制

<!-- 目录放置在此处 -->

Ionic Framework 遵循 <a href="https://semver.org/" target="_blank">语义化版本控制（SemVer）</a>规范：<code>主版本号.次版本号.修订号</code>。不兼容的 API 变更会递增 <code>主版本号</code>，添加向后兼容的功能会递增 <code>次版本号</code>，向后兼容的错误修复会递增 <code>修订号</code>。

## 发布计划

### 主版本发布

当 API 中引入破坏性变更时，将发布主版本。主版本大约每 **6 个月** 发布一次，可能包含破坏性变更。在主版本发布之前，会发布多个候选版本，以便在最终发布前收集反馈。候选版本将包含变更内容及原因的概述。

### 次版本发布

当新增功能或引入非破坏性的 API 变更时，将发布次版本。我们会严格测试所有变更以确保版本稳定，但新代码仍可能带来新问题。如果有任何功能或 API 变更，我们计划 **每月** 发布一个次版本。

### 修订版本发布

当包含错误修复、但 API 未变更且未引入破坏性变更时，将发布修订版本。我们计划 **每两周** 发布一个新的修订版本，但有时可能需要提前或延后发布。为确保修订版本能够修复现有代码而不引入新功能带来的问题，修订版本总是会在次版本发布之前发布。

## 更新日志

要查看 Ionic 的所有重要变更列表，请参阅 <a href="https://github.com/ionic-team/ionic/blob/master/CHANGELOG.md" target="_blank">更新日志</a>。其中包含按版本排序的所有错误修复和新功能列表。
